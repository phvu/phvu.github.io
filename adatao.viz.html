adatao.viz.adaviz.box_plot(data, category, **kwargs)[source]¶

Boxplot creates box plot charts. It requires one or two category fields, and one measurement field. There are two ways to invoke the charttype function - either with a category / measurement field couple, in which case the client will do the aggregation, or by specifying the quartiles, median, and possibly minimum and maximum. In that case aggregation happens before plotting ie at the server level.

Here are the parameters in input that have an impact on boxplot.

Parameters:category – (required) field used to group the records before examining their distribution.

Controls the name of the columns. :param category2: (optional) field used to group the records before examining their distribution. Controles the colors and name of the series. :param crossbar: (optional) if set to true, only the quartiles and medians are shown, there won’t be whiskers to minimum or maximum. :param dots: (optional) if set to true, dots for each of the actual record will be plotted. :param max: (optional) specifies the name of the field where to find the maximum value of a column. :param measurement: (required for the client aggregation method) what field of the dataset on which the distribution will be computed. :param median: (required for the server aggregation method) where to find the median value of a column. :param min: (optional) specifies the name of the field where to find the minimum value of a column :param q1: (required for the server aggregation method) where to find the 1st quartile value for a column :param q3: (required for the server aggregation method) where to find the 3rd quartile value for a column :return:

adatao.viz.adaviz.multiple_bar_chart(data, category, category2, measurement, **kwargs)[source]¶

Handle the small multiples situation for bar charts. In other terms, multiple bars draws several “mini-charts” in the space of a regular chart. Most of its functionality is dedicated to fit as many mini charts as needed in that space, utilize that space as well as possible, and if the charts can fit, and if it is allowed to do so, expand the space of the chart to fit everything. If multiple bars isn’t allowed to expand, it chooses the most relevant data to display.

How the multiple bars work: First it determines how many mini charts it can fit horizontally, based on label lengths and minChartWidth. Then, it determines how many charts it can fit vertically, based on minChartHeight, number of rows per chart, label heights, spacing etc. If it cannot fit all mini charts, it will drop rows of the mini charts to reduce them to their minChartHeight.

For instance, if the minChartHeight is 100, and a row needs 16 pixels, it will drop any row after the 6th. Then, it will only fit as many mini charts as possible. However, once that max number of mini charts has been reached, if there is still space available at the bottom, it will try to get those dropped rows back. If the chart is allowed to expand, then its height will increase to fit all minicharts and all rows. :param data: a pandas DataFrame :param category: indicates which field of the dataset will be used as a dimension to plot the

bars - this is the other axis relative to measurement.
Parameters:
  • category2 – indicates which field of the dataset will be used to group the bars in distinct mini charts.
  • measurement – indicates which field of the dataset will be used as a numerical measure - that will dictate the length of the bars.
  • kwargs – additional options: - title: provides a title for the chart. - minChartWidth indicates the minimum width for the charts in pixels. When resized, the charts can go up and down in width, but they won’t get under minChartWidth, which is set by default to 100. - minChartHeight indicates the minimum height for the charts in pixels. When resized, the charts can go up and down in height, but they won’t get under minChartHeight, which is set by default to 100. - canResize: (True/False) whether the chart is allowed to expand vertically or whether it must try to fit everything in a fixed space.
Returns:

depending on the renderer mode

It is possible to create a grouped bar chart and a stacked bar chart by setting the chart type as “bar.grouped” and “bar.stacked” respectively.

It is also possible to set a “grouped” or a “stacked” flag to true. There is also a possible “bar.histogram” chart type (which is not currently used), and a “histogram” flag that can be set to true.

Module contents¶

Adatao’s visualization engine

This sub-package implements a Python interface for adaViz - Adatao’s visualization engine.

For a complete documentation, check adaVizJS doc at https://github.com/adatao/adaVIZ-JS/blob/master/docs/spec.md

By default, all the plotting functions in this package return a bunch of JSON code, intended to be used inside Narratives.

For interactive works, you might want to output them to HTML files. This can be controlled using the viz.set_renderer_mode() function:

>>> viz.set_renderer_mode('html', output_dir='[your_local_path]')

This will make all functions give their output as a HTML files stored in the directory specified in output_dir. The HTML file name will be the timestamp at which the function is called. Every call to a function will return a single file. If output_dir is not given, it is defaulted to be the temporary folder on your system.

>>> viz.set_renderer_mode('browser')

Similar to html, but writes the output to the temporary folder and automatically opens a new browser window to show the result.

>>> viz.set_renderer_mode('narratives')

To be used inside Narratives.
